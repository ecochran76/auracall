# Policy | Turn Closeout

## Policy

- End-of-turn closeout should be compact and should use inline labels instead
  of standalone section headers when the content is short.
- Default order: `Status + Verification`, `Plan + Audit`, optional
  `Risks/Blockers`, then `Best Recommendation (Primary)`.
- `Best Recommendation (Primary)` must be the final sentence or paragraph.
- Include `Risks/Blockers` only when there are material blockers or actionable
  risks with exact unblockers.
- Include `Ranked Alternatives` only when there is genuine uncertainty or a
  material tradeoff; place it before the final recommendation.
- Omit labels that would only report that there are no issues; favor useful,
  actionable information and recommendations.
- Alternate closeout modes should be explicit and limited to cases like plan,
  audit, next-slice details, or roadmap reassessment.
- Do not end with vague “what do you want next?” language when a best recommendation is available.
## Adoption Notes

Use this module when the repo values consistent turn endings and explicit next-step guidance.
